## âœ… **Task 2: Deploy Apache and NGINX using Docker Compose**

### ğŸ§  Goal:

* Run **Apache** on port **91** and **NGINX** on port **92** using Docker Compose.

---

### ğŸ› ï¸ Step-by-Step Instructions:

#### Step 1: Create project directory

```bash
mkdir ~/grapevine-compose
cd ~/grapevine-compose
```

#### Step 2: Create `docker-compose.yml`

```yaml
version: '3'
services:
  apache:
    image: httpd:latest
    ports:
      - "91:80"
    container_name: apache-container

  nginx:
    image: nginx:latest
    ports:
      - "92:80"
    container_name: nginx-container
```

Save it as `docker-compose.yml` in `~/grapevine-compose`.

#### Step 3: Launch containers

```bash
docker-compose up -d
```

#### Step 4: Test it

Open these URLs in your browser:

* Apache â†’ [http://localhost:91](http://localhost:91)
* NGINX â†’ [http://localhost:92](http://localhost:92)

You should see the default pages of each web server.

## âœ… **Task 3: Initialize Docker Swarm and deploy Ubuntu containers in overlay network**

### ğŸ§  Goal:

* Create a **Docker Swarm cluster**.
* Deploy **2 Ubuntu containers** in a custom **overlay network**.
* Demonstrate they can **ping** each other.

---

### ğŸ› ï¸ Step-by-Step Instructions:

#### Step 1: Initialize Docker Swarm (on manager node)

```bash
docker swarm init --advertise-addr <MANAGER-IP>
```

> Replace `<MANAGER-IP>` with the actual IP of your machine.

#### Step 2: Create an overlay network

```bash
docker network create \
  --driver overlay \
  --attachable \
  grapevine-overlay
```

#### Step 3: Deploy Ubuntu service #1

```bash
docker service create \
  --name ubuntu1 \
  --network grapevine-overlay \
  --replicas 1 \
  ubuntu \
  sleep 10000
```

#### Step 4: Deploy Ubuntu service #2

```bash
docker service create \
  --name ubuntu2 \
  --network grapevine-overlay \
  --replicas 1 \
  ubuntu \
  sleep 10000
```

#### Step 5: Exec into one container and ping the other

1. Find container ID:

```bash
docker ps
```

2. Enter one container:

```bash
docker exec -it <ubuntu1-container-id> bash
```

3. Inside the container, run:

```bash
ping ubuntu2
```

You should see successful ping replies â€” verifying that the **overlay network allows cross-container communication**.
